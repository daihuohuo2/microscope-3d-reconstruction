## 1. 快速入门

### 1.1 系统要求

| 项目 | 要求 |
|-----|------|
| 操作系统 | Windows 10 / 11（64位）|
| Python | 3.8 或更高版本（推荐 3.9/3.10）|
| 相机 | 支持 MVS SDK 的海康工业相机 |
| 运动平台 | Z 轴电动平台（串口通信，Marlin 固件）|

---

### 1.2 环境配置（首次使用）

#### 第一步：克隆仓库

```bash
git clone https://github.com/daihuohuo2/code.git
cd code
```

> 如无 git，也可在 GitHub 页面点击 **Code → Download ZIP** 下载后解压。

---

#### 第二步：安装 Python

前往 [python.org](https://www.python.org/downloads/) 下载 Python 3.8 或更高版本（推荐 3.9/3.10），安装时勾选 **"Add Python to PATH"**。

---

#### 第三步：安装 Python 依赖库

在项目目录下执行：

```bash
pip install -r requirements.txt
```

如 pip 下载缓慢，换国内镜像：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

`requirements.txt` 包含以下库：

| 库 | 用途 |
|----|------|
| `PyQt5` | GUI 界面框架 |
| `numpy` | 图像矩阵运算 |
| `matplotlib` | 3D 点云可视化、深度图显示 |
| `pyserial` | 串口通信（Z 轴控制）|
| `opencv-python` | 图像处理、文件保存 |

---

#### 第四步：安装海康 MVS SDK

1. 前往 [海康机器人官网](https://www.hikrobotics.com/) 下载 **MVS 工业相机客户端软件**
2. 按默认路径安装（程序会自动搜索 DLL 路径）
3. 安装完成后将相机通过 USB 或 GigE 连接到电脑
4. 在 MVS 客户端中确认相机可以正常枚举和取流

> **注意**：本程序依赖 MVS SDK 的 `MvCameraControl.dll`，必须安装 MVS 才能控制相机。

---

#### 第五步：串口驱动（Z 轴运动平台）

1. 将运动平台通过 USB 转串口线连接到电脑
2. 安装对应的串口驱动（通常为 **CH340** 或 **CP2102**，驱动可在淘宝购买页或芯片官网下载）
3. 打开设备管理器，确认串口号（如 COM3）
4. 确保运动平台控制板已烧录 **Marlin 固件**，波特率默认 `19200`

---

#### 第六步：运行程序

```bash
python main.py
```

首次运行后会在项目目录自动生成 `setting.ini`，保存串口号、保存路径、像素/mm 等设置，下次启动自动恢复。

---

### 1.3 最简操作流程（5 分钟上手）

```
1. 枚举设备 → 选择相机 → 打开设备 → 开始取流
2. 刷新串口 → 选择串口 → 连接串口
3. 点击菜单栏「一键出图(I)」
4. 设置保存路径 → 设置 Z 轴范围 → 点击"一键出图"
5. 完成！查看自动保存的全焦图、深度图文件
```

> **菜单说明**：菜单栏有四个直接入口，单击即触发，无二级子菜单：
> - **点云重建(3)...** — 完整三维点云重建
> - **连续扫描重建(S)...** — 高速连续扫描
> - **一键出图(I)** — 快速全焦成像
> - **可编程拍摄(P)** — CSV 定时任务拍摄

## 6. 可编程拍摄

> **适用场景**：按 CSV 时间表自动拍摄延时序列，可逐张设置是否自动对焦、曝光、增益和灯光亮度，结束后自动合成 24 帧/s MP4 视频。

### 6.1 打开功能

```
点击菜单栏「可编程拍摄(P)」
```

### 6.2 CSV 任务格式

第一列为拍照时间，支持两种格式：
- **14 位**（含秒）：`YYYYMMDDHHmmss`，例如 `20260504143000`
- **12 位**（无秒）：`YYYYMMDDHHmm`，例如 `202605041430`

**已过期的时间会立即执行**，无需设置为未来时间。

CSV 文件编码推荐 GBK（Windows 中文系统保存 CSV 的默认编码），UTF-8 也兼容。

```csv
拍照时间,自动对焦,快门us,增益,灯光亮度
20260504143000,1,20000,2,180
20260504143100,0,5000,0,120
20260504143200,1,1200000,12,300
```

| 列 | 含义 | 有效范围 | 超范围处理 |
|---|------|---------|-----------|
| 1 | 拍照时间 | `YYYYMMDDHHmmss`（14位）或 `YYYYMMDDHHmm`（12位） | 格式错误会报错；已过期立即执行 |
| 2 | 是否自动对焦 | `1` 自动 / `0` 直接拍摄 | 非 1 均按直接拍摄理解 |
| 3 | 快门时间 | 10-1000000 μs | 小于 10 按 10，大于 1000000 按 1000000 |
| 4 | 增益 | 0-10 | 小于 0 按 0，大于 10 按 10 |
| 5 | 灯光亮度 | 0-255 | 小于 0 按 0，大于 255 按 255 |

### 6.3 操作步骤

| 步骤 | 操作 | 说明 |
|-----|------|------|
| 1 | 打开相机并开始取流 | 拍摄保存依赖实时取流 |
| 2 | 连接串口 | 用于控制灯光；需要自动对焦时也用于 Z 轴移动 |
| 3 | 点击菜单栏 **「可编程拍摄(P)」** | 直接弹出对话框 |
| 4 | 点击 **CSV 任务文件 / 浏览...** | 选择 CSV 文件 |
| 5 | 点击 **图片保存文件夹 / 浏览...** | 所有图片和视频会保存在这里 |
| 6 | 点击 **开始执行** | 系统按本机系统时间等待并执行 |
| 7 | 等待完成 | 自动生成 `output_24fps.mp4` |

### 6.4 执行规则

- 等待任务期间灯光保持关闭。
- 到达每行拍照时间后，系统设置快门、增益，并按该行亮度打开灯光。
- 如果该行自动对焦为 `1`，自动对焦会在这次亮灯窗口内完成，然后拍照。
- 灯光从打开时刻开始累计保持 20 秒，之后自动关闭。
- 每张图片命名为 `prog_序号_YYYYMMDD_HHMMSS.bmp`。
- 所有任务结束后，本次成功拍摄的图片会按文件名排序合成为 `output_24fps.mp4`。

## 7. 参数说明

### 7.1 扫描参数

| 参数 | 单位 | 建议范围 | 影响 |
|-----|------|---------|------|
| Z 步长 | mm | 0.05-0.3 | 越小越精确，但越慢 |
| 每步延时 | s | 0.1-0.5 | 确保平台稳定 |
| 扫描速度 | mm/s | 0.5-2.0 | （连续扫描）越快越快 |
| 采帧间隔 | s | 0.05-0.2 | （连续扫描）越小帧数越多 |

### 7.2 点云参数

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| Z 轴缩放系数 | 1.0 | 调整深度方向的比例 |
| 最小锐度阈值 | 20.0 | 低于此值的像素被过滤，建议 10-100 |

### 7.3 控制与算法功能

| 功能 | 实现方式 | 使用位置 |
|-----|---------|---------|
| 上位机控制 | PyQt5 图形界面，多线程执行扫描、融合、保存，避免界面卡顿 | 主窗口、三维重建、一键出图、可编程拍摄 |
| 下位机协同 | 通过串口发送 G-code 控制 Z 轴、归零、相对/绝对移动和 PWM 灯光；停拍扫描会等待 `M400` 完成后采帧 | 串口控制、点云重建、一键出图、连续扫描 |
| 底噪扣除 | 遮光后点击 **"采集底噪帧"**，系统采集 50 帧并计算逐像素均值模板；启用后每帧实时扣除并钳位到有效灰度范围 | 主窗口右侧 **"底噪扣除"** |
| HDR 增强 | 勾选 **"启用 HDR"** 后，系统在实时预览显示前对每帧做局部对比度增强；优先使用 OpenCV CLAHE，缺少 OpenCV 时自动退回百分位拉伸 | 主窗口左侧 **"HDR增强"** |
| 自动标定 | 采集移动前后两帧，通过亚像素相位相关计算位移，自动写入 `pixels/mm` 和 `μm/px` | 主窗口 **"自动标定"** |
| 快速比例尺 | 对白底黑点标定板做 blob 检测，随机取 5 个圆点并计算最近邻平均像素间距；按 200 μm 实际点距换算像素当量，右下角显示白色 μm 比例尺 | 主窗口 **"快速比例尺"**、输出文件 |
| 自动对焦 | 使用 **ATLAS Focus**：保持当前曝光/增益参数不变，对中央或双击 ROI 做有效纹理块筛选，融合多尺度 Tenengrad、Laplacian 与 Brenner 锐度评分；Z 轴先粗扫定位峰区，再按 0.030/0.015/0.008/0.004 mm 逐级自适应精扫，最终用二次拟合计算亚步长焦点位置，并在峰值不明显时给出低置信提示。**双击预览框中任意位置**可对该点进行 ROI 对焦；点击"自动对焦"按钮则恢复全图模式 | 主窗口 **"自动对焦"** 按钮 / 预览框**双击** |
| DFF 焦点深度恢复 | 对图像栈逐像素计算拉普拉斯锐度，选择最佳焦面生成深度图和全焦图，并用相邻三层二次拟合提升深度精度 | 点云重建、连续扫描重建、一键出图 |
| 三维点云映射 | 根据深度图、`pixels/mm`、`z_scale` 和锐度阈值生成 X/Y/Z 点云，导出 PLY/OBJ/CSV 并附带 RGB 纹理颜色 | 三维重建输出 |

### 7.4 参数调整建议

#### 场景 1：扫描速度优先

```
Z 步长: 0.2-0.3 mm
每步延时: 0.1-0.15 s
最小锐度阈值: 10-15
```

#### 场景 2：精度优先

```
Z 步长: 0.05-0.1 mm
每步延时: 0.2-0.3 s
最小锐度阈值: 30-50
```

#### 场景 3：点云质量差

```
↓ 减小 Z 步长（0.05 mm）
↓ 增加每步延时（0.25 s）
↓ 降低最小锐度阈值（10-15）
↓ 检查相机曝光是否合适
```

#### 场景 4：预览纹理不明显

```
1. 打开设备并开始取流
2. 在主窗口左侧「HDR增强」中勾选「启用 HDR」
3. 优先用于白底低对比样品、裂纹、颗粒边界和弱纹理观察
4. 若画面噪点被放大，先降低增益或补充光照，再按需要关闭 HDR
```

| 类型 | 格式 | 内容 | 用途 |
|-----|------|------|------|
| 全焦合成图 | PNG/TIFF | PNG 为 8 位灰度并带比例尺条；TIFF 保留 8 位或 12 位源数据且不叠加标注 | 测量、发表、归档 |
| 深度图 | 16 位 TIFF | 像素值为相对高度，单位 μm | ImageJ 剖面分析 |
| 点云数据 | PLY/OBJ | X/Y/Z、强度和 RGB 颜色 | 3D 软件查看、尺测量、体积计算 |
| 点云数据 | CSV | 表格数据 | Excel/脚本分析 |
| 输出清单 | JSON | 时间戳、参数、单位和文件列表 | 结果追溯 |

### 8.2 文件命名规则

## 9. 故障排除

### 9.1 相机问题

| 症状 | 可能原因 | 解决方法 |
|-----|---------|---------|
| 枚举不到设备 | 相机未连接/未供电 | 检查 USB/电源连接 |
| 打开设备失败 | SDK 未安装 | 安装相机厂商 SDK |
| 画面全黑 | 曝光时间过短 | 增加曝光时间 |
| 画面全白 | 曝光时间过长/光线过强 | 减少曝光或遮光 |
| 画面模糊 | 焦点不对/震动 | 调整焦距，稳定平台 |

### 9.2 串口问题

| 症状 | 可能原因 | 解决方法 |
|-----|---------|---------|
| 刷新不到串口 | 驱动未安装/未连接 | 安装 CH340/CP2102 驱动 |
| 连接失败 | 波特率错误 | 优先尝试 19200，再尝试 9600/115200 |
| Z 轴不动 | 串口选择错误 | 尝试其他串口 |
| Z 轴运动异常 | 平台未使能 | 检查平台开关/限位 |

### 9.3 扫描问题

| 症状 | 可能原因 | 解决方法 |
|-----|---------|---------|
| 扫描中断 | 超时/丢帧 | 增加延时，检查连接 |
| 合成图模糊 | 步长过大 | 减小 Z 步长 |
| 点云空洞多 | 锐度阈值过高 | 降低最小锐度阈值 |
| 点云噪点多 | 锐度阈值过低 | 提高最小锐度阈值 |
| 深度不准确 | 扫描范围设置错误 | 调整 Z 轴范围 |

### 9.4 性能问题

| 症状 | 可能原因 | 解决方法 |
|-----|---------|---------|
| 扫描很慢 | 步长太小/延时太长 | 增大步长，减少延时 |
| 可视化卡顿 | 点数太多 | 软件自动采样（限制 6 万点） |
| 内存不足 | 帧数太多/分辨率太高 | 减小扫描范围或降低相机分辨率 |

## 附录 A：快捷操作参考

### 一键出图（最快）

```
1. 确认相机取流 + 串口连接
2. 点击菜单栏「一键出图(I)」
3. 设置保存路径，Z 高位=1.0, Z 低位=-1.0, 步长=0.15
4. 点击"一键出图"
5. 1-2 分钟后完成
```

### 完整点云重建

```
1. 确认相机取流 + 串口连接
2. 点击菜单栏「点云重建(3)...」
3. 设置保存路径，Z 起始=-1.0, Z 结束=1.0, 步长=0.1
4. 点击"开始重建"
5. 完成后点击"可视化点云"
6. 点击"导出点云"保存
```

### 可编程拍摄

```
1. 准备 CSV：时间(14位或12位),自动对焦,快门us,增益,灯光亮度
2. 确认相机取流 + 串口连接
3. 点击菜单栏「可编程拍摄(P)」
4. 选择 CSV 和图片保存文件夹
5. 点击"开始执行"
6. 完成后查看 output_24fps.mp4
```
